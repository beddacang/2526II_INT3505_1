## 2. Các bước thực hiện

### Bước 1: Chuẩn bị môi trường
* Ngôn ngữ: Python 3.14
* Thư viện: `pytest` (Testing), `requests` (API Call), `locust` (Performance).
* Lệnh cài đặt: `pip install pytest requests locust`

### Bước 2: Kiểm thử tự động (Automated API Testing)
Sử dụng **Pytest** để kiểm tra endpoint: `GET /users/1`
* Kiểm tra Status Code (Mong đợi: 200).
* Kiểm tra cấu trúc dữ liệu JSON (Schema validation).
* Kiểm tra thời gian phản hồi đơn lẻ.

### Bước 3: Kiểm thử hiệu năng (Performance Testing)
Sử dụng **Locust** để giả lập tải trọng:
* Số lượng người dùng giả lập: 20 VUs.
* Tốc độ tăng trưởng (Spawn rate): 2 users/s.
* Thời gian chạy: ~2 phút.

## 4. Phân tích và Kết luận (QA Insights)
1.  **Độ tin cậy:** Hệ thống đạt độ ổn định tuyệt đối (0% lỗi) dưới tải trọng 20 người dùng. Các ràng buộc về logic dữ liệu được đảm bảo 100%.
2.  **Hiệu năng:** Thời gian phản hồi p95 ở mức 43s là rất cao so với tiêu chuẩn ngành (thường < 2s). 
    * *Nguyên nhân:* Do độ trễ đường truyền mạng từ máy khách tới server public.
    * *Khuyến nghị:* Cần triển khai các cơ chế bộ nhớ đệm (Caching) hoặc tối ưu hóa truy vấn ở phía Server để giảm p95 xuống dưới 500ms.
