Bước 1: Kiểm thử tích hợp tự động (Automated Integration Test)
Đây là bước kiểm tra xem API có chạy đúng logic không.

Mở Terminal trong VS Code (đảm bảo đã cd vào thư mục Tuan_08/Code_demo).

Chạy lệnh: ```powershell
python -m pytest test_integration.py --verbose

Xem kết quả: Kết quả hiển thị ngay tại Terminal.

Nếu hiện PASSED: Thành công.

Nếu hiện FAILED: Đọc dòng báo lỗi màu đỏ ngay bên dưới để biết lỗi tại đâu (ví dụ: do mạng chậm nên Response Time bị quá 0.5s).

Bước 2: Kiểm thử hiệu năng (Performance Test)
Đây là bước giả lập nhiều người dùng để đo sức chịu tải.

Chạy lệnh khởi động engine:

PowerShell
python -m locust -f locustfile.py
Truy cập giao diện web: Mở trình duyệt (Chrome/Edge) và nhập đường link:

Link: http://localhost:8089

Cấu hình thông số trên Web:

Number of users: 20 (Số người dùng giả lập).

Spawn rate: 2 (Mỗi giây thêm 2 người cho đến khi đủ 20).

Host: https://jsonplaceholder.typicode.com (Địa chỉ API cần test).

Bắt đầu: Nhấn nút Start swarming.

Bước 3: Theo dõi và Phân tích kết quả thực tế
Khi Locust đang chạy, bạn cần quan sát các Tab trên trình duyệt:

Tab Statistics (Thống kê): Xem các con số nhảy liên tục về RPS (số yêu cầu mỗi giây) và Failures (số lần lỗi).

Tab Charts (Biểu đồ): Xem đường biểu đồ thời gian phản hồi. Nếu đường này đi ngang là hệ thống ổn định, nếu đi lên dốc đứng là hệ thống đang "quá tải".
