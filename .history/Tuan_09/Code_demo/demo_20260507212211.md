# Demo Tuần 09: API Versioning
## Cách chạy
1. Cài đặt thư viện: `pip install fastapi uvicorn`
2. Chạy server: `uvicorn app:app --reload`
3. Truy cập: `http://127.0.0.1:8000/docs`

## Các kỹ thuật triển khai
- **URL Path Versioning**: `/api/v1/users` và `/api/v2/users`.
- **Lifecycle Management**: Sử dụng Header `Deprecation` và `Sunset`.




Hướng dẫn triển khai API Versioning với FastAPI
1. Cài đặt môi trường
Chúng ta sẽ sử dụng FastAPI và Uvicorn (một ASGI server nhanh nhất cho Python).

Bash
pip install fastapi uvicorn
2. Cấu trúc thư mục chuyên nghiệp
Trong Python, việc tách biệt các phiên bản bằng Sub-applications hoặc APIRouter là cách làm tốt nhất:

Plaintext
project/
├── main.py          # Điểm khởi chạy ứng dụng
├── api/
│   ├── v1/
│   │   └── endpoints.py
│   └── v2/
│       └── endpoints.py
3. Logic triển khai chi tiết
V1 (Legacy): Sử dụng các Model dữ liệu cũ. Chúng ta sẽ "ép" thêm các Header vào Response để báo hiệu việc sắp đóng cửa API.

V2 (Breaking Change): Thay đổi cấu trúc Model (ví dụ: dùng Pydantic để định nghĩa lại schema dữ liệu).

Lifecycle Management: Sử dụng Middleware hoặc Dependencies để tự động chèn các thông tin Sunset (ngày tắt) và Deprecation.

Giải thích chi tiết các thành phần (Dành cho tài liệu học tập)
1. Tại sao dùng Pydantic (BaseModel)?
Trong Python, khi bạn thay đổi từ UserV1 sang UserV2, Pydantic sẽ tự động kiểm tra dữ liệu. Nếu bạn trả về thiếu trường is_active ở bản V2, FastAPI sẽ báo lỗi ngay lập tức. Điều này giúp bạn kiểm soát Breaking Changes cực tốt ngay từ phía Server.

2. Ý nghĩa của các Header trong V1
Deprecation: Thông báo bằng mã máy rằng endpoint này không nên được dùng cho các dự án mới nữa.

Sunset: Đây là "hạn chót". Nó giúp phía khách hàng (Client) có kế hoạch ngân sách và thời gian để nâng cấp code của họ.

Link: Chỉ dẫn họ đến nơi có tài liệu mới (v2).

3. URL Path vs Header Versioning
URL Path (/v1/): Giúp bạn có thể chạy song song hai bộ tài liệu khác nhau. FastAPI sẽ tạo ra 2 phần riêng biệt trong Swagger UI.

Header (X-API-Version): Giúp API trông chuyên nghiệp và gọn gàng hơn, nhưng khó debug hơn vì không thể nhìn trực tiếp vào thanh địa chỉ trình duyệt để biết đang dùng bản nào.

4. Mã lỗi 410 (Gone)
Khác với 404 (Không tìm thấy), mã 410 khẳng định rằng: "Tài nguyên này từng tồn tại, nhưng giờ đã bị xóa vĩnh viễn". Điều này giúp các lập trình viên phía Client biết rằng họ bắt buộc phải sửa code chứ không phải do gõ sai địa chỉ.

🧪 Cách chạy và Kiểm tra
Lưu mã nguồn vào file main.py.

Chạy lệnh: python main.py.
