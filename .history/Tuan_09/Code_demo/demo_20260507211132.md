Hướng dẫn triển khai API Versioning & Lifecycle
1. Chuẩn bị môi trường
Đầu tiên, khởi tạo dự án Node.js và cài đặt thư viện express:

Bash
mkdir api-versioning-demo
cd api-versioning-demo
npm init -y
npm install express
2. Cấu trúc thư mục (Best Practice)
Để quản lý nhiều phiên bản mà không làm code trở thành "một đống mì ống" (spaghetti code), chúng ta phân chia theo thư mục:

src/controllers/: Chứa logic xử lý của từng phiên bản.

src/routes/: Định nghĩa các endpoint cho v1, v2.

src/app.js: Điểm vào chính của ứng dụng.

3. Các bước thực hiện trong Code
Bước 1: Định nghĩa Controller
V1: Trả về dữ liệu cũ (dễ gây lỗi nếu client kỳ vọng cấu trúc mới).

V2: Trả về dữ liệu đã được cấu trúc lại (Breaking Change).

Bước 2: Thiết lập Header Lifecycle
Trong bản V1, chúng ta sử dụng Middleware để đính kèm Header Deprecation và Sunset. Điều này cực kỳ quan trọng để thông báo cho các bên tích hợp rằng họ cần nâng cấp.

Bước 3: Điều hướng (Routing)
Sử dụng Express Router để tách biệt hoàn toàn /v1 và /v2. Ngoài ra, chúng ta cài đặt thêm một Header-based Versioning để hỗ trợ các Client muốn giữ URL sạch.
